LED Logger - User Manual

Operator guide for the Windows executable build.

Helios via WebSocket/API Novastar COEX via SNMP + Traps Remote Monitor (TCP 8090)

1. Package Contents

  • Required: LED_Logger.exe
  • Required: config.json (or config.example.json as template)
  • Auto-created: history.json (saved sessions)

Keep these files in the same folder.

2. System Requirements

  • Windows 10/11
  • Network access to LED processors
  • Firewall rules for:
    • TCP 8090 (remote monitor page)
    • UDP 10162 (COEX SNMP traps)
    • UDP 161 outbound (COEX SNMP polling)

3. First Start

  1. Launch LED_Logger.exe.
  2. If no devices exist, Device Management opens automatically.
  3. Add devices manually or run SCAN NETWORK.
  4. Click SAVE and CLOSE.
  5. Watch events in LIVE MONITOR.

4. Device Types

  • Helios
  • Novastar_COEX

UI labels are typically HELIOS and COEX.

5. Configuration Example

{
  "processors": [
    {
      "name": "Main Helios",
      "ip": "192.168.1.10",
      "type": "Helios"
    },
    {
      "name": "MX2000 Main",
      "ip": "192.168.1.20",
      "type": "Novastar_COEX",
      "snmp_community": "public"
    }
  ],
  "web_auth": {
    "username": "admin",
    "password_hash": "03ac674216f3e15c761ee1a5e255f067953623c8b388b4459e13f978d7c846f4"
  }
}

Important Fields

  • name: shown in UI and logs
  • ip: processor IP address
  • type: Helios or Novastar_COEX

COEX Optional Fields

  • snmp_community (usually public)
  • coex_backup_api_enabled
  • coex_backup_api_port (default 8001)
  • coex_backup_api_poll_interval
  • coex_backup_api_log_every_poll

Default web login is admin / 1234 until changed in settings.

6. App Workflow

LIVE MONITOR Columns

  • Time, Device, MAC, OPT, PORT, TILE, Message, OID

Color Meaning

Red: error / fault
Orange: warning
Green: recover / OK
Gray: info / system

History and Baselines

  • CLEAR LOG / SAVE SESSION stores current session into history.json.
  • A fresh baseline starts immediately after clearing.

7. Remote Monitor Webpage

The app starts a webserver on port 8090 and shows the URL in the sidebar.

Example: http://192.168.0.50:8090

  • Basic Auth protected
  • Auto-refresh enabled
  • Shows latest logs and global device status

To change credentials: CONFIGURE DEVICES > Web Interface Login.

8. COEX SNMP Traps and Auto-Config

  • Health polling over SNMP
  • Trap listener on UDP 10162

On first online detection, the app attempts to auto-configure trap target settings on the COEX device.

If this fails due to firmware limitations or SNMP write permissions, a manual VMP instruction is logged.

9. Troubleshooting Checklist

No remote monitor page

  • Check if port 8090 is free.
  • Try http://localhost:8090 on the logger PC.
  • Verify firewall permissions.

COEX stays offline

  • Ping the controller.
  • Check SNMP community value.
  • Ensure SNMP is enabled on the device.

No trap events

  • Allow UDP 10162 inbound on the logger PC.
  • Verify trap target in VMP.
